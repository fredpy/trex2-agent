Detailed Description

template<class AbstractProduct, class Id, class ConsArg, class ProductRef = AbstractProduct *, class IdComp = std::less<Id>>
class TREX::utils::Factory< AbstractProduct, Id, ConsArg, ProductRef, IdComp >

Generic global factory.

Template Parameters:
AbstractProductBase type of the products
IdIdentifier type for the producers
ConsArgType of the argument passed for the production of the new product.
ProductRefThe type returned by producer after creation
IdCompOrdering functor between Id

This class implements the factory design pattern. More specifically class is used as a singleton class that provides a global factory that any can use to produce objects derived from AbstractProduct and having a constructor with ConsArg as argument. The access to producers is given by a key of type Id.

The general idea of this class was to allow automatic registering and un-registering of the producers during their construction and destruction. Producers can be simply defined using the Factory::declare template class. For example in a Factory that produces abstract instances of Foo which is the mother class of Bar that can be constructed by passing an int as argument you can uses the following code to declare a producer for Bar associating it to the string "bar 

 # include "Factory.hh"

 TREX::utils::Factory<Foo, std::string, int>::declare<Bar> 
      bar_decl("bar");

As long as bar_decl is existing the factory will provide an access to this producer. To be able to use it one can do : 

 SingletonUse< Factory<Foo,std::string, int> > bar_fact;
 Foo *var = bar_fact->get("bar")(5);
See also:
TREX::utils::SingletonUse
Precondition:
ProductRef is expected to be a pointer like type for AbstractProduct
IdComp should be a complete order functor other Id
Note:
In order to ease the automatioc registration of producers the factory class is implemented as a Singleton. This allows the producers to know directly how to find their associated Factory. On the other hand you can't have easily multiple factories of the same type.
As for now Factory class relies on std::map to store the producers. As a result we need to have a complete order for Id. It may be better to switch in the future on C++ TR1 collections that would only require the Id type to provide hash and equality functor allowing incomplete order.
