Detailed Description

template<class Ty>
class TREX::utils::SharedVar< Ty >

multi-thread shared variable

This class implements a simple utility to manage a variable which is manipulated by multiple threads. It ensures that this variable is accessed only by the thread which have reserved (locked) it.

Template Parameters:
TyType of the variable to be shared
Precondition:
Ty must provides a copy constructor
Ty must be default constructible

template<class Ty>
typedef boost::unique_lock< SharedVar<Ty> > TREX::utils::SharedVar< Ty >::scoped_lock

Scoped variable locking type.

This type can be used to locally lock one SharedVar based in a C++ scope. A new instance of this class will lock the mutex associated to the SharedVar during its whiole lifetime and release the mutex on its destruction.

This provide a simple, exception safe, way to lock a SharedVar during a critical section while avoiding to have it remains locked in the eventuality of an exception.

For example on can implement the function getval(SharedVar const &) that returns the value of a SharedVar like this: 

 template<class Ty>
 Ty getval(SharedVar<Ty> const &var) {
   SharedVar<Ty>::scoped_lock slock(var);
   return *var;
 }

At the begining of this function the creation of slock locks the mutex associated to var. This alows to access to var on the next line. Finally, as we leave the function context slock will be destroyed and by doing so will release the mutex it locked during its creation.
